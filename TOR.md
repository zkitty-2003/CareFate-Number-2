# Terms of Reference (TOR) - โครงการ CareFate

## 1. วัตถุประสงค์
เพื่อกำหนดขอบเขตและรายละเอียดการทำงานของระบบบันทึกและติดตามสุขภาพอัจฉริยะ CareFate ให้มีความชัดเจน ครอบคลุมฟังก์ชันการใช้งานของผู้ใช้ทุกช่วงวันและผู้ดูแลระบบ

## 2. ขอบเขตการทำงานของ "ผู้ใช้งานทั่วไป" (User Functional Requirements)

| ฟังก์ชันการทำงาน | รายละเอียด | หมายเหตุ | วันที่เสร็จ |
| :--- | :--- | :--- | :--- |
| **1. ระบบลงทะเบียนและเข้าสู่ระบบ** | ☐ 1.1 ลงทะเบียนด้วยอีเมลและรหัสผ่าน<br>☐ 1.2 มีการตรวจสอบรูปแบบอีเมล (Email Regex Validation) โดยระบบจะไม่อนุญาตให้ลงทะเบียนหากรูปแบบไม่ถูกต้อง<br>☐ 1.3 ข้อกำหนดรหัสผ่าน: ความยาว 6-20 ตัวอักษร, ต้องมีตัวพิมพ์ใหญ่ (A-Z) อย่างน้อย 1 ตัว, ต้องมีตัวพิมพ์เล็ก (a-z) อย่างน้อย 1 ตัว และห้ามมีตัวอักษรพิเศษ<br>☐ 1.4 การกรอกข้อมูลส่วนตัวเบื้องต้น: ชื่อผู้ใช้ (Username) และ วัน/เดือน/ปีเกิด (Birth Date)<br>☐ 1.5 ระบบส่งลิงก์ยืนยันตัวตน (Email Verification) ไปยังอีเมลของผู้ใช้ทันทีหลังจากลงทะเบียนเสร็จสิ้น<br>☐ 1.6 ระบบส่งลิงก์รีเซ็ตรหัสผ่าน (Forgot Password) ไปยังอีเมลเมื่อผู้ใช้กดขอความช่วยเหลือ<br>☐ 1.7 เมื่อผู้ใช้คลิกลิงก์จากอีเมล ระบบจะต้องนำผู้ใช้กลับมาสู่หน้า "เปลี่ยนรหัสผ่าน" (Reset Password Page) บนแอปพลิเคชันโดยอัตโนมัติ | | |
| **2. ระบบจัดการโปรไฟล์และธีม (Personalization)** | ☐ 2.1 ระบบคำนวณอายุอัตโนมัติจากวันเกิดเพื่อแนะนำธีมเริ่มต้น (Youth, Working, Elder)<br>☐ 2.2 ผู้ใช้งานสามารถสลับเปลี่ยนธีมได้ตามความต้องการผ่านหน้าการตั้งค่า<br>☐ 2.3 การปรับแต่ง UI: เมื่อเปลี่ยนธีม ระบบจะเปลี่ยนสีหลัก (Primary Color), ชุดไอคอน และขนาดฟอนต์ให้เหมาะสมกับช่วงวัยโดยอัตโนมัติ | | |
| **3. ระบบบันทึกกิจกรรมสุขภาพ (12 Features)** | ☐ 3.1 **การจัดการบันทึกยา (Medication Management)**: ผู้ใช้งานสามารถลงข้อมูลยาที่จำเป็นต้องทานได้อย่างละเอียด ดังนี้:<br> - บันทึกชื่อยาและประเภทของยา (เช่น ยาเม็ด, ยาน้ำ)<br> - ระบุขนาดและปริมาณที่ต้องทานในแต่ละครั้ง<br> - กำหนดความถี่ (รายวัน/รายสัปดาห์) และระบุช่วงเวลาที่ทาน (ก่อนอาหาร/หลังอาหาร)<br> - ระบบตั้งเวลาแจ้งเตือน (Reminder) แบบเจาะจงเวลา เพื่อป้องกันการลืมทานยา<br> - ผู้ใช้งานสามารถตรวจสอบประวัติการทานยาย้อนหลังและสถานะการทานยาในแต่ละวันได้<br>☐ 3.2 **บันทึกการนอน (Sleep)**: บันทึกเวลาเข้านอน, เวลาตื่น และการประเมินคุณภาพการนอน (1-5 ดาว)<br>☐ 3.3 **บันทึกมื้ออาหาร (Food)**: เลือกมื้ออาหาร (เช้า, กลางวัน, เย็น, ทานเล่น), ระบุชื่อเมนู และปริมาณแคลอรี่<br>☐ 3.4 **ดวงสุขภาพ (Horoscope)**: ระบบสุ่มคำทำนายสุขภาพรายวันเพื่อให้ความบันเทิงและคำแนะนำเบื้องต้น<br>☐ 3.5 **ตั้งเป้าหมาย (Goals)**: สร้างเป้าหมายรายวัน/รายสัปดาห์ (เช่น เป้าหมายการเรียน/ชีวิต), ระบุประเภท และเครื่องหมายขีดฆ่าเมื่อทำสำเร็จ<br>☐ 3.6 **การออกกำลังกาย (Exercise)**: ระบุประเภทกิจกรรม, ระยะเวลา (นาที) และคำนวณแคลอรี่ที่เผาผลาญเบื้องต้น<br>☐ 3.7 **บันทึกรอบเดือน (Period)**: บันทึกวันที่เริ่ม/สิ้นสุด, ระบุระดับความเจ็บปวด และอาการข้างเคียง<br>☐ 3.8 **การดูแลพาหนะ (Vehicle)**: บันทึกเลขไมล์ปัจจุบัน, วันที่ตรวจเช็คล่าสุด และแจ้งเตือนรอบการบำรุงรักษาถัดไป (สำหรับวัยทำงาน)<br>☐ 3.9 **บันทึกการขับถ่าย (Excretion)**: บันทึกเวลา, ลักษณะ (ปกติ, ท้องผูก, ท้องเสีย) และความถี่ (เน้นสำหรับผู้สูงอายุ)<br>☐ 3.10 **การนัดหมายแพทย์ (Appointment)**: บันทึกชื่อแพทย์, โรงพยาบาล, แผนก และตั้งค่าแจ้งเตือนล่วงหน้า 1 วัน/1 ชม.<br>☐ 3.11 **ประวัติสุขภาพ (History)**: แสดงรายการบันทึกกิจกรรมย้อนหลังในรูปแบบรายการ (Log List) โดยมีฟังก์ชันหลัก ได้แก่:<br> - การสลับแท็บระหว่าง "บันทึกสุขภาพ" และ "ประวัติการแชท"<br> - **ผู้ใช้งานสามารถเลือกกรองรายการบันทึกเพื่อแสดงผลตามช่วงเวลาที่ต้องการ** (ตัวเลือก: ทั้งหมด, วันนี้, 7 วันล่าสุด)<br> - ระบบตัวกรองเลือกตามหมวดหมู่กิจกรรม (ทุกหมวดหมู่)<br> - การแสดงผลรายการบันทึกประกอบด้วย ไอคอนหมวดหมู่, ชื่อกิจกรรม (เช่น บันทึกรอบเดือน), รายละเอียดเบื้องต้น และวันเวลาที่บันทึก<br>☐ 3.12 **การตั้งค่า (Settings)**: แก้ไขข้อมูลส่วนตัว, ปรับโหมดการแสดงผล (Light/Dark Mode) และเปลี่ยนภาษา | มีการปรับปรุงและขยายขีดความสามารถการทำนายดวงในข้อ 3.4 เพิ่มเติม เพื่อแก้ปัญหาคำทำนายทางเดียว (One-way) และข้อจำกัดการสุ่มทั่วไป โดยสร้างฟังก์ชันระบบหมอดู AI อัจฉริยะแยกความเชี่ยวชาญ 5 ด้าน พร้อมระบบประเมินความพึงพอใจโดยละเอียด ตามข้อที่ 8.1, 8.2, 8.3 และ 8.4 | |
| **4. ระบบ AI Chatbot (AI Assistant)** | ☐ 4.1 **การสื่อสารอย่างเป็นธรรมชาติ**: ผู้ใช้งานสามารถพิมพ์โต้ตอบเป็นภาษาไทยได้เหมือนคุยกับคนจริงๆ โดย AI สามารถเข้าใจคำถามและคำสั่งต่างๆ ได้อย่างแม่นยำ<br>☐ 4.2 **การสนทนาที่ต่อเนื่อง**: ผู้ใช้งานสามารถคุยถาม-ตอบต่อเนื่องได้โดยไม่ต้องเริ่มเกริ่นใหม่ทุกครั้ง เพราะระบบสามารถจดจำเรื่องราวที่คุยค้างไว้ก่อนหน้าได้<br>☐ 4.3 **การปรึกษาเรื่องสุขภาพส่วนบุคคล**: ผู้ใช้งานสามารถสอบถามข้อมูลเกี่ยวกับประวัติสุขภาพของตนเองได้ (เช่น "เมื่อคืนนอนไปกี่ชั่วโมง" หรือ "วันนี้กินยาไปหรือยัง") โดย AI จะดึงข้อมูลจากบันทึกมาตอบให้ทันที<br>☐ 4.4 **การเลือกเพื่อนร่วมทางตามสไตล์**: ผู้ใช้งานสามารถเปลี่ยนบุคลิกของ AI ให้เข้ากับความชอบตามธีมที่เลือกได้ (เช่น คุยเป็นเพื่อนวัยรุ่น, เป็นเลขาส่วนตัว, หรือเป็นหลานที่มาคอยดูแล)<br>☐ 4.5 **การรับรายงานสรุปสุขภาพอัจฉริยะ**: ผู้ใช้งานจะได้รับสรุปวิเคราะห์แนวโน้มสุขภาพรายสัปดาห์ พร้อมคำแนะนำที่เข้าใจง่ายและนำไปปรับใช้ได้จริง<br>☐ 4.6 **การรับคำแนะนำเชิงรุก**: ผู้ใช้งานจะได้รับการทักทายหรือการแจ้งเตือนจาก AI เมื่อระบบพบว่ามีข้อมูลสุขภาพที่สำคัญยังไม่ได้ถูกบันทึก<br>☐ 4.7 **การรับคำแนะนำด้านความปลอดภัย**: ผู้ใช้งานจะได้รับคำเตือนและคำแนะนำเบื้องต้นที่ปลอดภัยเสมอ โดยระบบจะแนะนำให้ปรึกษาแพทย์ในกรณีที่มีอาการรุนแรง | | |
| **5. ระบบการแจ้งเตือน (Notification System)** | ☐ 5.1 **การตั้งค่าการแจ้งเตือนตามใจชอบ**: ผู้ใช้งานสามารถเลือกเปิดหรือปิดการแจ้งเตือนในแต่ละหมวดหมู่กิจกรรมได้ (เช่น แจ้งเตือนยาอย่างเดียว หรือแจ้งเตือนมื้ออาหารด้วย)<br>☐ 5.2 **การกำหนดเวลาแจ้งเตือนที่แม่นยำ**: ผู้ใช้งานสามารถเลือกตั้งเวลาที่ต้องการให้ระบบแจ้งเตือนได้แบบรายชั่วโมงและรายนาที<br>☐ 5.3 **ระบบเตือนซ้ำ (Smart Recurring)**: ผู้ใช้งานสามารถตั้งค่าให้ระบบแจ้งเตือนซ้ำทุกวัน หรือเฉพาะบางวันในระดับสัปดาห์ได้<br>☐ 5.4 **การแจ้งเตือนก่อนเวลากิจกรรม**: ผู้ใช้งานสามารถรับการแจ้งเตือนล่วงหน้า (เช่น เตือนนัดแพทย์ล่วงหน้า 1 ชั่วโมง) เพื่อเตรียมตัวก่อนถึงเวลาจริง<br>☐ 5.5 **การตอบสนองผ่านการแจ้งเตือน**: ผู้ใช้งานได้รับข้อความแจ้งเตือนที่ระบุชื่อกิจกรรมอย่างชัดเจน (เช่น "ถึงเวลาทานยาแก้แพ้แล้วค่ะ") เพื่อการใช้งานที่สะดวกและรวดเร็ว | | |

## 3. ขอบเขตการทำงานของ "ผู้ดูแลระบบ" (Administrator Functional Requirements)

| ฟังก์ชันการทำงาน | รายละเอียด | หมายเหตุ | วันที่เสร็จ |
| :--- | :--- | :--- | :--- |
| **1. การบันทึกและจัดการบัญชีผู้ใช้** | ☐ 1.1 **การเข้าถึงระบบที่ปลอดภัย**: ผู้ดูแลระบบสามารถเข้าสู่ระบบด้วยสิทธิ์แอดมิน (Admin Authentication) แยกจากผู้ใช้ทั่วไป<br>☐ 1.2 **การตรวจสอบสถานะผู้ใช้**: ผู้ดูแลระบบสามารถค้นหาและตรวจสอบรายละเอียดพื้นฐานของผู้ใช้งาน (ชื่อ, อีเมล, วันที่สมัคร) ได้ทั้งหมด<br>☐ 1.3 **การควบคุมสิทธิ์การใช้งาน**: ผู้ดูแลระบบสามารถระงับการใช้งาน (Suspend) หรือเปิดใช้งานบัญชีผู้ใช้ที่ต้องการได้<br>☐ 1.4 **การจัดการรหัสผ่าน**: ผู้ดูแลระบบสามารถดำเนินการรีเซ็ตรหัสผ่าน (Reset Password) ให้กับผู้ใช้งานที่ร้องขอหรือลืมรหัสผ่านได้<br>☐ 1.5 **การดูแลความเรียบร้อยของข้อมูล**: ผู้ดูแลระบบสามารถลบบัญชีผู้ใช้ที่ผิดกฎระเบียบหรือไม่ได้ใช้งานเป็นเวลานานออกจากระบบได้ (โดยผู้ใช้ที่ถูกลบเนื่องจากไม่ได้ใช้งานสามารถกลับมาสมัครใหม่ได้ แต่อีเมลที่ทำผิดกฎจะถูกบันทึกเป็น Blacklist เพื่อป้องกันการสมัครซ้ำ) | | |
| **2. ระบบวิเคราะห์และรายงานสถิติ** | ☐ 2.1 **การวิเคราะห์กลุ่มเป้าหมาย**: ผู้ดูแลระบบสามารถดูสถิติแยกตามช่วงอายุ (Youth, Working, Elder) เพื่อนำไปวิเคราะห์การตลาดหรือกลุ่มผู้ใช้หลัก<br>☐ 2.2 **การติดตามความนิยมของธีม**: ผู้ดูแลระบบสามารถดูสัดส่วนการเลือกใช้งานธีมต่างๆ เพื่อประเมินความพึงพอใจด้าน UI<br>☐ 2.3 **การวิเคราะห์พฤติกรรมการใช้งาน**: ผู้ดูแลระบบสามารถตรวจสอบได้ว่าฟีเจอร์ใดใน 12 ฟีเจอร์ที่มีการใช้งานมากที่สุดในแต่ละวัน<br>☐ 2.4 **การติดตามช่วงเวลาการใช้งาน (Peak Time)**: ผู้ดูแลระบบสามารถดูช่วงเวลาที่มีผู้ใช้งานเข้ามาบันทึกข้อมูลหนาแน่นที่สุด เพื่อเตรียมพร้อมด้าน Server<br>☐ 2.5 **การเฝ้าดูการทำงานของ AI**: ผู้ดูแลระบบสามารถตรวจสอบปริมาณการใช้งาน Chatbot เพื่อประเมินประสิทธิภาพของระบบ AI | | |
| **3. การจัดการข้อมูลและการส่งออกรายงาน** | ☐ 3.1 **การส่งออกข้อมูลผู้ใช้**: ผู้ดูแลระบบสามารถส่งออกรายการผู้ใช้งานในรูปแบบไฟล์ที่รองรับ (เช่น CSV/Excel) เพื่อการสำรองข้อมูล<br>☐ 3.2 **การสรุปรายงานสุขภาพภาพรวม**: ผู้ดูแลระบบสามารถดึงข้อมูลสรุปสถิติสุขภาพแบบไม่ระบุตัวตน เพื่อนำไปใช้เป็นรายงานผลการดำเนินงานโครงการ<br>☐ 3.3 **การตรวจสอบ Log ระบบ**: ผู้ดูแลระบบสามารถตรวจสอบประวัติการเข้าใช้งานระบบบันทึกความปลอดภัย | | |

## 4. เทคโนโลยีที่ใช้ (Technical Stack)
- **Frontend**: HTML5, CSS3 (Vanilla CSS), JavaScript (Vanilla JS)
- **Backend/API**: Python (FastAPI/Flask)
- **Database & Auth**: Supabase (PostgreSQL)
- **AI Integration**: AI Model สำหรับ Chatbot และการสรุปผล
- **Hosting**: Local Server / Cloud Platform

## 5. ภาคผนวก: การปรับปรุงระบบความปลอดภัยและการยืนยันตัวตน (Security & UX Verification Upgrades)

### 5.1 ปัญหาที่พบในระบบแบบเดิม (Problems Identified)
1. **ปัญหาช่องโหว่การสแปมส่งเมลซ้ำ (Email Spam / Bombing Vulnerability)**: ในการลงทะเบียนแบบทั่วไป หากผู้ประสงค์ร้ายรู้อีเมลเป้าหมาย แล้วนำไปพิมพ์ลงทะเบียนซ้ำในระบบ จะสามารถกดปุ่มเพื่อกระตุ้นให้เซิร์ฟเวอร์ส่งอีเมลยืนยันซ้ำ ๆ ไปรบกวนถล่มกล่องจดหมายของเป้าหมายได้โดยที่เป้าหมายไม่ได้สมัครใจ
2. **ปัญหาลิงก์ยืนยันตัวตนหมดอายุเกิน 24 ชั่วโมง (Link Expiration)**: เมื่อลิงก์แรกสุดหมดอายุการใช้งานหลังจาก 24 ชั่วโมง ผู้ใช้จริงที่ต้องการกลับมาเปิดใช้งานอีเมลเดิมจะไม่สามารถสมัครใหม่ได้เนื่องจากระบบจะบล็อกว่า *"อีเมลถูกใช้งานแล้ว"* ส่งผลให้เกิดปัญหาทางด้านประสบการณ์ผู้ใช้ (UX)

### 5.2 ทำไมจึงจำเป็นต้องดำเนินการแก้ไข (Rationale for Action)
- เพื่อยกระดับความปลอดภัยทางด้านเครือข่ายและการจัดส่งข้อมูลของระบบ ป้องกันไม่ให้แอปพลิเคชันของเราถูกนำไปใช้เป็นเครื่องมือสแปมอีเมลส่งความรำคาญ
- เพื่อลดโอกาสเกิดข้อผิดพลาดในการประมวลผลและการจัดส่งอีเมล (Rate Limiting Error)
- เพื่อปรับปรุงประสบการณ์ของผู้ใช้ (UX/UI) ให้มีความสมเหตุสมผล ชัดเจน และมีทางออกในการกู้คืนอีเมลเดิมอย่างปลอดภัย

### 5.3 สิ่งที่ดำเนินการแก้ไขและปรับปรุงเสร็จสิ้น (Completed Implementations)
- [x] **จำกัดอายุผู้สมัคร 13 ปีขึ้นไป (13+ Age Limit)**: แสดงข้อความแจ้งเตือนสีแดงพรีเมียม `อายุ 13+` ในช่องวันเกิด และเพิ่มการบล็อกลงทะเบียนหน้าเว็บทันทีหากผู้กรอกอายุต่ำกว่า 13 ปี
- [x] **ปรับปรุงแถบข้อความเตือนเมื่ออีเมลซ้ำ (UX Error Message)**: เปลี่ยนข้อความเมื่อกดสมัครด้วยอีเมลซ้ำให้เป็นคำแนะนำเชิงช่วยเหลือ: *"อีเมลถูกใช้งานแล้ว หากเป็นอีเมลของคุณแต่ยังไม่ได้กดยืนยันตัวตน กรุณาตรวจสอบกล่องข้อความหรือกล่องจดหมายขยะเพื่อกดยืนยันก่อนเข้าใช้งาน"*
- [x] **ระบบส่งลิงก์ยืนยันตัวตนใหม่หลังผ่านด่านรหัสผ่านเท่านั้น (Secure Resend System)**: 
  - บล็อกฟังก์ชันการขอรับลิงก์ใหม่จากหน้าลงทะเบียน
  - ให้ผู้ใช้นำอีเมลที่ลิงก์หมดอายุไปกรอกเข้าสู่ระบบที่ **หน้าล็อกอิน (Login Screen)**
  - ระบบจะส่งลิงก์ยืนยันตัวตนฉบับใหม่ไปให้ **เฉพาะกรณีที่ผู้ใช้กรอกรหัสผ่าน (Password) ได้ถูกต้องเท่านั้น** เพื่อสกัดกั้นการสแปมจากบุคคลภายนอก 100% (เนื่องจากผู้อื่นไม่มีทางรู้รหัสผ่านของเรา)
  - กำหนดให้ลิงก์เก่ากลายเป็นโมฆะ (Invalidated) ทันทีที่มีการขอลิงก์ใหม่ล่าสุด เพื่อความมั่นใจในความปลอดภัยสูงสุดของข้อมูลผู้ใช้งาน

## 6. ภาคผนวก: การปรับปรุงและขยายฟังก์ชันระบบหมอดู AI อัจฉริยะและการประเมินความพึงพอใจ (AI Fortune Teller & Interactive Review Upgrades)

### 6.1 ปัญหาที่พบในระบบแบบเดิม (Problems Identified)
1. **ข้อจำกัดในการสุ่มดวงชะตา (Static Randomizing Limitation)**: ระบบเดิมทำได้เพียงการสุ่มคำทำนายด้านสุขภาพทั่วไปแสดงผลทางเดียว (One-way) ไม่มีตัวเลือกดวงชะตาเฉพาะด้านที่ผู้ใช้ต้องการทราบ
2. **การขาดมิติการปฏิสัมพันธ์และการประเมินผล (Lack of Feedback Loop)**: ผู้ใช้งานไม่มีช่องทางในการลงคะแนนความพึงพอใจหรือเขียนคำวิจารณ์ต่อคำทำนายของนักพยากรณ์ AI
3. **การจัดระเบียบข้อมูลสถิติที่แออัด (Data Visualization Density)**: เมื่อมีจำนวนผู้ใช้รีวิวเพิ่มขึ้น ข้อมูลการรีวิวสะสมในระบบมีความกระจัดกระจายและแออัด ยากต่อการนำเสนอผลคะแนนสรุปอย่างเป็นระบบ

### 6.2 ทำไมจึงจำเป็นต้องดำเนินการแก้ไข (Rationale for Action)
- เพื่อยกระดับความสามารถในการพยากรณ์ด้วย AI ให้มีความละเอียดและตรงตามความต้องการของผู้ใช้งานแต่ละบุคคลมากยิ่งขึ้น
- เพื่อสร้างช่องทางการประเมินผลความพึงพอใจ (Rating & Review Feedback) ที่มีความชัดเจนและโปร่งใสตามหลักสากล
- เพื่อนำข้อมูลสถิติและคะแนนเรตติ้งมานำเสนอในรูปแบบ Dashboard สรุปผลและแผนภูมิแท่งกราฟิก (Rating Breakdown) ที่อ่านง่าย สวยงาม และสามารถนำข้อมูลไปต่อยอดการประเมินความพึงพอใจระบบได้จริง

### 6.3 สิ่งที่ดำเนินการแก้ไขและปรับปรุงเสร็จสิ้น (Completed Implementations)
- [x] **8.1 ระบบสุ่มคำทำนายดวงชะตาและสุขภาพรายวันแยกตามความเชี่ยวชาญของหมอดู AI (AI Fortune Teller Categorization)**: เพิ่มตัวเลือกหมอดู AI รวม 5 ท่าน แยกประเภทตามความเชี่ยวชาญเฉพาะด้าน (ความรัก, การเงิน, การงาน, สุขภาพ, ภาพรวมชีวิต)
- [x] **8.2 ระบบประเมินผลคะแนนเรตติ้งและรีวิวความคิดเห็น (Interactive Rating & Review System)**: เพิ่มฟังก์ชันให้ผู้ใช้ร่วมลงคะแนนสะสม (1-5 ดาว) และแสดงความคิดเห็นต่อนักพยากรณ์แต่ละคน เพื่อสร้างปฏิสัมพันธ์สองทาง (Two-way Interaction)
- [x] **8.3 หน้าต่างสรุปสถิตินักพยากรณ์และฟิลเตอร์อัจฉริยะ (Oracle Statistics Dashboard with Filter)**: พัฒนาหน้าสรุปผลที่รองรับการกรองแท็บจัดกลุ่มรีวิวอย่างเป็นอิสระ (แยกตามรายหมอดู หรือแยกตามระดับคะแนนดาว)
- [x] **8.4 ระบบแผนภูมิสถิติแท่งแจกแจงสัดส่วนดาวรีวิว (Interactive Rating Breakdown)**: พัฒนาตัวนำเสนอข้อมูลสถิติด้วยแผนภูมิกราฟแท่งแบบไดนามิกเพื่อสะท้อนความนิยมและความเชื่อถือของหมอดูแต่ละท่านอย่างเข้าใจง่าย
